## Patrón de software

Para este prototipo se aplicó el patrón de 3 capas, complementado con el enfoque MVC en la capa de presentación, siguiendo las buenas prácticas vistas en la Semana 06. La capa de presentación gestiona la interacción con el usuario mediante vistas web o móviles, controladores REST y modelos de datos (DTO), permitiendo registrar productos, ventas y servicios de forma intuitiva. La capa de aplicación o dominio centraliza las reglas del negocio: control de stock, cálculo de IGV, margen de ganancia en servicios, registro de compras y ventas, así como la generación de boletas y reportes periódicos. En esta capa se aplican principios de encapsulamiento y separación de responsabilidades, garantizando que cada servicio cumpla una función específica. La capa de datos o persistencia usa repositorios o DAO para realizar operaciones seguras sobre una base de datos relacional, facilitando pruebas unitarias y cambios de motor sin afectar la lógica principal. Este patrón mejora la mantenibilidad, asegura la consistencia de las transacciones y permite escalar el sistema hacia APIs o microservicios en el futuro. Además, su estructura modular hace más sencilla la depuración y la incorporación de nuevas funcionalidades, como reportes personalizados o validaciones automáticas de boletas.
