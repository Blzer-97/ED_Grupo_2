## Patrón de software

El patrón de 3 capas separa la presentación, la lógica de negocio y los datos, lo que facilita el mantenimiento y las pruebas del sistema. En la capa de presentación, se aplica el patrón MVC, donde las vistas permiten la interacción del usuario (ingreso de productos, generación de boletas), los controladores gestionan las peticiones y el modelo transporta los datos. La capa de aplicación agrupa las reglas principales del negocio, como el cálculo del IGV, el margen de ganancia en servicios, la validación del stock, el costo promedio y la generación de reportes. La capa de datos maneja la persistencia mediante repositorios o DAO, lo que permite realizar pruebas y cambiar de motor de base de datos fácilmente. Este patrón acelera el desarrollo del prototipo, mantiene la organización del código y asegura la coherencia en las transacciones, permitiendo escalar el sistema en el futuro si se requiere.
