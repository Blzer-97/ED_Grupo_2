## Patrón del diseño de software (IDE de Desarrollo)

En la capa de presentación se implementará el enfoque MVC utilizando JavaFX 17 para construir una interfaz gráfica dinámica e intuitiva que permita registrar productos, gestionar ventas y visualizar reportes mediante componentes como javafx.chart. La capa de aplicación concentrará las reglas del negocio, incluyendo el cálculo de IGV, el control de stock y la generación de boletas, mientras que la capa de datos gestionará la persistencia a través de repositorios conectados a una base de datos no relacional. Todo el sistema se desarrollará en Java SE 17 LTS, aprovechando su robustez y soporte a largo plazo. Se empleará el IDE Apache NetBeans 17 por su integración con JavaFX y su entorno visual amigable . 
## Patrón de la arquitectura del software

Para este prototipo se aplicó el patrón de 3 capas, complementado con el enfoque MVC en la capa de presentación. La capa de presentación gestiona la interacción con el usuario mediante vistas web o móviles, controladores REST y modelos de datos (DTO), permitiendo registrar productos, ventas y servicios de forma intuitiva. La capa de aplicación o dominio centraliza las reglas del negocio: control de stock, cálculo de IGV, margen de ganancia en servicios, registro de compras y ventas, así como la generación de boletas y reportes periódicos. En esta capa se aplican principios de encapsulamiento y separación de responsabilidades, garantizando que cada servicio cumpla una función específica. La capa de datos o persistencia usa repositorios o DAO para realizar operaciones seguras sobre una base de datos relacional, facilitando pruebas unitarias y cambios de motor sin afectar la lógica principal. Este patrón mejora la mantenibilidad y asegura la consistencia de las transacciones. Además, su estructura modular hace más sencilla la incorporación de nuevas funcionalidades, como reportes personalizados o validaciones automáticas de boletas.
